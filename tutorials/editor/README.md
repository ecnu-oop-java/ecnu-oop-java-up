## 1. 什么是 Editor

### 1.1 Editor 的定义

Editor（编辑器）是用于编写和修改文本文件的软件。  
当我们写程序时，代码本质上也是文本，所以可以用 editor 编写代码。

如果把写代码类比成写论文：

1. 记事本像最原始的纯文本输入工具；
2. editor 像更专业的写作工具，能高亮、补全、格式化、跳转；
3. IDE（Integrated Development Environment，集成开发环境）则是在 editor 基础上进一步整合编译、调试、测试、项目管理等能力。

### 1.2 为什么需要使用 Editor

理论上，你可以用记事本写 Java；实际开发中，这样效率非常低，也很容易出错。  
使用 editor 的主要原因有：

1. **语法高亮**：不同关键字、字符串、注释有不同颜色，更容易阅读；
2. **自动补全**：输入类名、方法名时能自动提示，减少拼写错误；
3. **错误提示**：在编译前就能发现很多明显问题；
4. **格式化代码**：让代码风格统一、易于维护；
5. **项目管理**：便于打开整个工程，而不是一个个文件单独修改；
6. **调试支持**：可以设置断点，逐步查看程序执行过程；
7. **插件扩展**：能按语言和项目类型增加功能。

### 1.3 Editor、IDE、插件、工作区

这些概念在后续会经常出现：

1. **Editor**：主要负责编辑代码；
2. **IDE**：在 editor 基础上集成更多开发能力；
3. **Plugin / Extension（插件）**：给 editor 增加新功能；
4. **Workspace（工作区）**：你当前在 editor 中打开的项目目录；
5. **Terminal（终端）**：在 editor 内部直接执行命令行命令的窗口。

对初学者来说，可以先把 VS Code 理解为一个“通过插件扩展后非常强大的 editor”。

## 2. 常见的 Editor

课程中主要使用 **VS Code**。下面是几种常见工具的简要说明。

### 2.1 VS Code

特点：

1. 轻量、跨平台、安装简单；
2. 插件生态丰富；
3. 对 Git、Markdown、终端、多语言开发支持都很好；
4. 很适合课程教学和多门课共用一个工具链。

适合人群：

1. 初学者；
2. 需要同时写 Java、Markdown、Git 文档的同学；
3. 希望工具简单、上手快的同学。

### 2.2 IntelliJ IDEA

特点：

1. Java 支持非常强；
2. 重构、导航、项目分析能力很强；
3. 功能更完整，但占用资源也更高。

适合人群：

1. 主要做 Java 开发；
2. 后续打算长期做 Java/Spring 项目。

### 2.3 Eclipse

特点：

1. Java 领域历史很长；
2. 插件体系成熟；
3. 传统课程或老项目中仍然常见。

不足：

1. 界面和交互相对老旧；
2. 对初学者来说不如 VS Code 直观。

### 2.4 简单结论

如果你的目标是完成本课程、写 Java 项目、写 Markdown 文档、配合 Git 管理代码，那么 **VS Code 是最合适的默认选择**。  
它不是 Java 专用工具，但对教学和课程项目来说，平衡性最好。

## 3. VS Code 的下载安装

### 3.1 下载地址

VS Code 官方网站：

<https://code.visualstudio.com/>

建议：

1. 优先从官网下载；
2. 选择 **Stable** 版本；
3. 不建议随意从第三方网站下载。

### 3.2 Windows 安装

步骤：

1. 打开官网下载页面；
2. 选择 Windows 版本安装包；
3. 运行安装程序；
4. 安装过程中建议勾选：
   `Add to PATH`
   `Open with Code`
   `Add "Open with Code" action to Windows Explorer`
5. 安装完成后启动 VS Code。

### 3.3 macOS 安装

步骤：

1. 打开官网下载页面；
2. 下载 macOS 安装包；
3. 将 VS Code 拖入 `Applications`；
4. 首次打开时允许系统安全提示；
5. 后续可从 Dock 或应用程序目录启动。

### 3.4 Linux 安装

步骤：

1. 打开官网下载页面；
2. 根据发行版选择 `.deb` 或 `.rpm`；
3. 使用系统包管理器安装；
4. 安装后在应用菜单中打开 VS Code。

### 3.5 安装后第一次打开要做什么

第一次启动后，建议先完成这几件事：

1. 打开命令面板：`Ctrl+Shift+P` 或 `Cmd+Shift+P`；
2. 熟悉左侧活动栏：资源管理器、搜索、源代码管理、运行与调试、插件；
3. 打开设置页，搜索常用配置；
4. 打开一个完整项目文件夹，而不是只打开单个 `.java` 文件。

## 4. VS Code 基本配置

### 4.1 为什么要做基本配置

默认配置并不一定适合写 Java。  
合理配置可以减少很多“工具问题”，让你把精力放在代码和实验内容上。

### 4.2 推荐的基础设置

课程场景下，建议至少关注这些设置：

1. 自动保存；
2. 文件编码为 UTF-8；
3. 关闭或弱化容易分散注意力的界面元素；
4. 配置终端；
5. 统一缩进与格式化行为。

### 4.3 推荐的编辑器界面习惯

建议：

1. 左侧资源管理器常开，方便查看项目结构；
2. 底部终端常用，便于直接运行 `java`、`javac`、`mvn`；
3. 不要同时打开太多无关插件侧栏；
4. 尽量使用“打开文件夹”而不是“打开单文件”。

### 4.4 推荐的工作方式

1. 打开项目根目录；
2. 先看目录结构；
3. 再安装 Java 插件；
4. 确认 JDK 检测成功；
5. 最后再开始写代码、运行、调试。

## 5. VS Code 编辑器配置

这一节主要讲和“写代码体验”直接相关的配置。

### 5.1 推荐配置项

1. **缩进**：Java 代码通常使用 4 个空格；
2. **格式化**：保存时自动格式化；
3. **导包整理**：保存时自动整理 `import`；
4. **行宽参考线**：帮助控制代码长度；
5. **字体与字号**：保证长时间阅读舒适；
6. **空白字符显示**：帮助发现缩进问题；
7. **括号高亮**：帮助理解代码结构。

### 5.2 推荐的 `settings.json` 示例

本目录下提供了一份示例配置文件：

[settings.json](settings.json)

你可以把其中的思路迁移到项目的 `.vscode/settings.json` 中。  
下面是这份配置解决的主要问题：

1. 统一缩进与编码；
2. 自动格式化和整理导包；
3. 让 Java 文件默认交给 Java 插件格式化；
4. 给终端和工作区提供更稳定的默认行为。

### 5.3 工作区设置和全局设置的区别

VS Code 常见有两类设置：

1. **用户设置（User Settings）**：影响你所有项目；
2. **工作区设置（Workspace Settings）**：只影响当前项目。

课程里更推荐：

1. 通用外观设置放用户设置；
2. Java 项目相关设置放工作区设置；
3. 不同课程、不同项目尽量不要共用一套“写死路径”的配置。

## 6. VS Code 插件配置

### 6.1 为什么 Java 在 VS Code 中需要插件

VS Code 默认并不自带完整 Java 能力。  
Java 的语法分析、调试、测试、项目管理等功能主要依赖插件。

### 6.2 必装插件

最推荐的方式是直接安装：

`Extension Pack for Java`

它通常会包含或关联下面这些核心插件：

1. `Language Support for Java by Red Hat`
2. `Debugger for Java`
3. `Test Runner for Java`
4. `Maven for Java`
5. `Project Manager for Java`

如果你只想记一个名字，记住 **`Extension Pack for Java`** 就够了。

### 6.3 可选插件

下面这些不是必须，但很实用：

1. `GitLens`：辅助理解 Git 历史；
2. `Markdown All in One`：方便同时编写实验文档；
3. `Chinese (Simplified) Language Pack`：如果你更适应中文界面。

### 6.4 插件安装步骤

步骤：

1. 打开 VS Code；
2. 点击左侧 `Extensions`；
3. 搜索 `Extension Pack for Java`；
4. 点击 `Install`；
5. 安装完成后重启或重新加载窗口。

### 6.5 插件配置建议

建议：

1. 不要一次安装过多“功能重复”的 Java 插件；
2. 不要同时启用多个 formatter；
3. 如果课程主要使用 Maven，优先保证 Maven 相关插件正常；
4. 插件安装后先看是否识别到 JDK，再开始写代码。
