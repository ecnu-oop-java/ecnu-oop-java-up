## 1. Markdown 是什么

Markdown 是一种轻量级标记语言，核心思想是：

1. 文本文件可直接阅读；
2. 用少量符号表达结构（标题、列表、代码、表格等）；
3. 便于版本管理（配合 Git 对比变更非常清晰）。

为什么课程里推荐使用 Markdown：

1. 写作速度快；
2. 跨平台；
3. 与代码仓库天然集成；
4. 能同时服务「老师阅读」和「你自己复习」。

## 3. 5 分钟写出第一份文档（最小可用）

新建 `README.md`，写入下面内容并预览：

````markdown
# Project 01

## 项目简介

这是一个使用 Java 实现的示例项目。

## 运行环境

- JDK 17
- Maven 3.9+

## 目录结构

```text
.
├── src/main/java
├── src/main/resources
├── imgs
└── README.md
```

## 运行结果

![运行截图](imgs/result.png)

## 核心代码

```java
public class Example {

    }

```

## 补充说明

更详细说明见 [docs/example.md](docs/example.md)。
````

你已经掌握了 70% 课程场景。

## 4. 核心语法（必须掌握）

### 4.1 标题（Heading）

语法：

```markdown
# 一级标题

## 二级标题

### 三级标题

#### 四级标题
```

规则：

1. 一个文档通常只用一个一级标题（`#`）；
2. 标题层级不要跳级（例如 `##` 后直接 `####`）；
3. `#` 后加一个空格再写标题文字。

### 4.2 段落与换行

语法：

```markdown
这是第一段。

这是第二段。
```

说明：

1. 空一行表示新段落；
2. 普通回车不会强制换行（依平台而定）；
3. 强制换行可用行尾两个空格，或 `<br>`（不推荐滥用 HTML）。

### 4.3 强调（粗体、斜体、删除线）

语法：

```markdown
**粗体**
_斜体_
**_粗斜体_**
~~删除线~~
```

建议：

1. 中文场景优先用 `**粗体**` 表示重点；
2. 删除线常用于展示“旧方案已废弃”。

### 4.4 引用（Blockquote）

语法：

```markdown
> 这是引用内容
>
> > 这是二级引用
```

常见用途：

1. 提示说明；
2. 结论摘要；
3. 注意事项。

### 4.5 列表（有序/无序）

语法：

```markdown
- 无序项 A
- 无序项 B
  - 子项 B1

1. 有序项 1
2. 有序项 2
3. 有序项 3
```

注意：

1. 同一层级的符号风格保持一致；
2. 子项缩进使用空格，不要混用 Tab。

### 4.6 代码（行内代码与代码块）

语法：

````markdown
行内代码：使用 `System.out.println()`。

```java
public static void main(String[] args) {
    System.out.println("hello");
}
```
````

建议：

1. 代码块尽量标注语言（如 `java`、`bash`、`json`）；
2. 只贴关键代码，完整代码用链接到源文件。

### 4.7 链接（Link）

语法：

```markdown
[显示文字](https://example.com)
[相对路径链接](docs/example.md)
[代码链接](src/main/java/Example.java)
```

建议：

1. 仓库内文档优先用相对路径；
2. 链接文字写清楚目的，不要全是“点击这里”。

### 4.8 图片（Image）

语法：

```markdown
![图片说明](imgs/logo.png)
```

建议：

1. 把图片统一放进 `imgs/`；
2. 文件名有意义，如 `q1-result.png`；
3. 控制图片大小，避免仓库臃肿；
4. 给图片写可读的替代文本（`[]` 内的文字）。

### 4.9 分隔线

语法（任选其一）：

```markdown
---
---
```

用途：章节切分。

### 4.10 表格（Table）

语法：

```markdown
| 功能     | 语法         | 示例     |
| -------- | ------------ | -------- |
| 粗体     | `**text**`   | **text** |
| 行内代码 | `` `code` `` | `code`   |
```

对齐：

```markdown
| 左对齐 | 居中 | 右对齐 |
| :----- | :--: | -----: |
| A      |  B   |      C |
```

### 4.11 转义字符

如果你想显示 Markdown 符号本身，用反斜杠 `\` 转义：

```markdown
\*这不是斜体\*
\# 这不是标题
```

### 4.12 HTML 混用（了解）

Markdown 支持部分 HTML，例如：

```html
<details>
	<summary>点击展开</summary>
	这里是隐藏内容。
</details>
```

原则：能不用 HTML 就不用，优先保持 Markdown 可读性。

## 5. GFM 扩展语法（课程常用）

以下语法在 GitHub/多数现代编辑器中可用，但在少数平台可能不支持。

### 5.1 任务列表（Task List）

```markdown
- [x] 完成需求分析
- [x] 编码
- [ ] 补充测试
```

### 5.2 自动链接

```markdown
https://github.com
```

### 5.3 脚注（Footnote）

```markdown
这是一个结论[^1]。

[^1]: 这里是脚注说明。
```

### 5.4 数学公式（视平台支持）

行内公式：

```markdown
$E = mc^2$
```

块级公式：

```markdown
$$
\sum_{i=1}^{n} i = \frac{n(n+1)}{2}
$$
```

## 7. 常见错误与排查

### 7.1 标题不显示

原因：`#` 后没空格。

错误：

```markdown
#标题
```

正确：

```markdown
# 标题
```

### 7.2 代码块没有高亮

原因：未声明语言。

错误：

````markdown
```
System.out.println("x");
```
````

正确：

````markdown
```java
System.out.println("x");
```
````

### 7.3 图片显示失败

排查顺序：

1. 路径是否写错（大小写也要一致）；
2. 图片是否真的在仓库里；
3. 是否用了本机绝对路径（应改成相对路径）。

### 7.4 列表层级混乱

原因：缩进不统一，或者中间缺空行。  
解决：统一用 2 或 4 个空格缩进，必要时在段落间加空行。

### 7.5 表格错位

原因：列数不一致。  
解决：保证每一行 `|` 数量对应同样的列数。

## 8. 写作规范（建议）

1. 一个文档只有一个 `#` 一级标题；
2. 标题名称尽量“名词化”，例如“运行方式”“测试结果”；
3. 每段只表达一个核心信息；
4. 代码与截图只放“支持结论所需”的最小内容；
5. 链接和图片优先相对路径，保证仓库迁移后仍可用；
6. 对外部参考资料给出来源链接；
7. 提交前在 VS Code 预览和 GitHub 页面各检查一次。
