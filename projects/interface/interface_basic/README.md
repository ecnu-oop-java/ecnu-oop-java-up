# 接口与抽象类基础

## 实验背景

你是一名农业科技公司的初级软件工程师。公司正在为现代化农场开发一套后台管理系统——“EcoFarm 1.0”。

现在农场里充满了各种不同的实体：

- **生物资产**：如奶牛、果树（需要生长周期，产出农产品）。
- **机械设备**：如自动收割机、无人机（需要耗电，需要维修）。
- **普通物资**：如化肥、饲料（主要属性是库存和保质期）。

系统需要统一管理这些截然不同的事物。例如，老板希望点击一个按钮就能算出“农场总资产价值”，或者点击一个按钮就能“执行所有设备的维护并喂养所有动物”。

## 实验内容

### 任务一

在“EcoFarm”系统中，任何存在于农场的东西都是一项“资产”。

#### 1.定义抽象类 `FarmAsset`

所有资产都有名称和基础价值，但计算价值的方式截然不同（例如：机器折旧算价值，动物按体重算价值）。

| **数据成员** | **类型/修饰符**      | **说明**                            |
| ------------- | -------------------- | ----------------------------------- |
| `name`        | `String` (protected) | 资产名称 (如 "Bessie", "Reaper-01") |
| `baseCost`    | `double` (protected) | 初始购买成本/基础价值               |

| **方法**                | **修饰符**        | **说明**                                          |
| ----------------------- | ----------------- | ------------------------------------------------- |
| `FarmAsset(name, cost)` | `public`          | 构造函数                                          |
| `getAssetInfo()`        | `public`          | 返回字符串，包含名称和基础成本                    |
| `getCurrentValue()`     | `public abstract` | 抽象方法。返回资产当前价值 (不同资产计算公式不同) |

#### 2.定义接口

农场中的对象不仅仅是资产，它们还有特定的“功能”。请定义以下接口：

**接口 `IMaintainable`**

   适用对象：动物（需要喂食）、机器（需要维修/充电）。

   | **方法**                 | **返回值** | **说明**                                           |
   | ------------------------ | ---------- | -------------------------------------------------- |
   | `performMaintenance()`   | `double`   | 执行维护动作，并返回本次维护产生的费用             |
   | `getMaintenanceDetail()` | `String`   | 返回维护内容的文本描述 (如 "更换机油", "喂食牧草") |

**接口 `IHarvestable`** 

   适用对象：动物（产奶/蛋）、农作物（产果实）。

   | **方法**          | **返回值** | **说明**                                            |
   | ----------------- | ---------- | --------------------------------------------------- |
   | `isHarvestable()` | `boolean`  | 判断当前状态是否可以收获                            |
   | `harvest()`       | `String`   | 执行收获动作，返回收获的具体物品名称 (如 "5L Milk") |

### 任务二

#### 1.实现木围栏 `WoodenFence` 类

围栏是农场的静止资产，它没有特殊功能（不需要接口），只涉及基础价值计算。

| **数据成员** | **类型/修饰符**    | **说明**             |
| ------------ | ------------------ | -------------------- |
| `length`     | `double` (private) | 围栏长度（单位：米） |

| **方法**            | **修饰符** | **说明**                                                     |
| ------------------- | ---------- | ------------------------------------------------------------ |
| `WoodenFence(...)`  | `public`   | 构造函数。需传入 `name`, `baseCost` (每米单价), `length`。 注意使用 `super(name, cost)` 初始化父类。 |
| `getCurrentValue()` | `public`   | 重写父类方法。 计算公式：`baseCost * length` (假设围栏不折旧)。 |

#### 2.实现苹果树 `AppleTree`  类

苹果树既是资产，又是可收获的作物。它需要实现 `IHarvestable` 接口。

| **数据成员** | **类型/修饰符**     | **说明**                     |
| ------------ | ------------------- | ---------------------------- |
| `age`        | `int` (private)     | 树龄（单位：年）             |
| `isWatered`  | `boolean` (private) | 是否已浇水。默认为 `false`。 |

| **方法**            | **修饰符** | **说明**                                                     |
| ------------------- | ---------- | ------------------------------------------------------------ |
| `AppleTree(...)`    | `public`   | 构造函数。初始化名称、基础成本和树龄。                       |
| `water()`           | `public`   | 自定义方法。将 `isWatered` 设为 `true`。                     |
| `getCurrentValue()` | `public`   | 重写父类方法。 计算公式：`baseCost + (age * 10)`。           |
| `isHarvestable()`   | `public`   | 实现接口方法。 逻辑：当 `age > 3` (3年以上结果) 且 `isWatered` 为 `true` 时返回 true。 |
| `harvest()`         | `public`   | 实现接口方法。 逻辑：若可收获，返回字符串 "Box of Apples"，并将 `isWatered` 重置为 `false`；否则返回 null。 |


#### 3.实现奶牛 `DairyCow` 类

奶牛是 `FarmAsset`，同时是 `IMaintainable` (吃草) 和 `IHarvestable` (产奶)。

| **数据成员** | **类型/修饰符**     | **说明**               |
| ------------ | ------------------- | ---------------------- |
| `weight`     | `double` (private)  | 奶牛体重 (kg)          |
| `isHungry`   | `boolean` (private) | 是否饥饿 (初始为 true) |

| **方法**               | **说明**     | **逻辑要求**                                                 |
| ---------------------- | ------------ | ------------------------------------------------------------ |
| `getCurrentValue()`    | 重写父类方法 | 价值 = `baseCost` + (`weight` * 15.0)                        |
| `performMaintenance()` | 实现接口方法 | 1. 每次维护费用为 `weight` * 0.1 2. 将 `isHungry` 设为 `false` |
| `harvest()`            | 实现接口方法 | 1. 如果 `isHungry` 为 true，返回 null 或 提示信息 2. 否则返回 "20L Milk"，并将 `isHungry` 设为 `true` (产奶后会饿) |
| `getAssetInfo()`       | 重写父类方法 | 在父类信息基础上增加体重信息                                 |

#### 4.实现智能收割机 `SmartReaper` 类

智能收割机继承自 `FarmAsset`，需要实现接口`IMaintainable` (不实现 `IHarvestable`)

| **数据成员**    | **类型/修饰符**    | **说明**                   |
| --------------- | ------------------ | -------------------------- |
| `efficiency`    | `double` (private) | 收割效率 (如 0.8 代表 80%) |
| `yearOfService` | `int` (private)    | 已服役年限                 |

| **方法**               | **说明**     | **逻辑要求**                                           |
| ---------------------- | ------------ | ------------------------------------------------------ |
| `getCurrentValue()`    | 重写父类方法 | 价值 = `baseCost` * (0.9 的 `yearOfService` 次方)      |
| `performMaintenance()` | 实现接口方法 | 费用固定为 150.0 (润滑与充电费)，打印 "Reaper charged" |
| `getAssetInfo()`       | 重写父类方法 | 增加显示效率和服役年限                                 |


### 任务三(vibe coding,使用ai编程助手辅助编写代码)

现在你手里有一堆零散的类，我们需要一个**总控系统**来统一管理它们。

#### 1.设计农场管家 `FarmManager` 

请根据以下业务需求文档，构建 `FarmManager` 类。

**业务需求文档：**

1. **资产库管理**：
   - 系统需要有一个“仓库”，能够存储不限数量、不限种类的农场资产。
   - 提供一个 `addAsset` 接口用于入库。
2. **资产盘点功能**：
   - 老板需要一张报表，列出仓库里所有物品的名称、基础成本以及当前估值。
   - 思考提示：不同物品计算估值的方式不一样，如何用一行统一的代码获取它们的当前价值？
3. **日常运营模拟**：
   - 这是农场每天必须执行的批处理任务。系统需要遍历仓库中的每一个物品，并根据物品的特性自动执行操作：
     - **对于需要供养/维护的物品**：自动扣除维护费，并记录维护日志。
     - **对于可收获的物品**：检查是否成熟。如果成熟，则执行收获，并记录收获了什么。
     - **对于普通静物（如围栏）**：跳过，不做任何操作。
   - 你的仓库数组里存的是通用的 `FarmAsset`，编译器并不知道谁能吃草，谁能产奶。你需要找到一种方法来动态识别每个物品具备的“能力”（接口），并安全地调用对应的方法。

#### 2.编写场景测试

请在 `Main` 类中模拟一个真实的“农场发家史”剧本，以验证你的系统是否健壮。

**场景脚本：**

1. **初始化**：创建管家对象。
2. **采购环节**：
   - 买入一头奶牛 "Bessie"（体重 500kg）。
   - 买入一台收割机 "Reaper-V1"。
   - 买入一颗苹果树（树龄 5 年，未浇水）。
   - 建一段 100 米长的木围栏。
3. **盘点环节**：打印所有资产信息。检查一下围栏有价值吗？奶牛的价值包含体重加成吗？
4. **运营环节（第一天）**：
   - 执行 `dailyProcess()`。
   - 预期结果：奶牛应该被喂食（产生费用）；收割机应该被维护（产生费用）；苹果树因为没浇水，不应该有收获；围栏应该被忽略。
5. **特殊事件**：
   - 手动给苹果树浇水（调用你之前写的 `water()` 方法）。
   - 再次执行 `dailyProcess()`。
   - 预期结果：这次苹果树应该能收获了。
