# 容器

请利用 java 容器完成以下问题。

对于每个问题，请先思考利用哪个容器可以更好的抽象问题。

## 实验内容

1. 你现在在管理一个具有实时收益的直播间，直播总时长为`n`分钟，第`i`分钟的收益可以表示为 $a_i$ 。作为直播间的管理员，你需要对一些时间段的总收益进行分析从而调整直播内容，所以你想快速的获得任意时间段 `[l, r]` 之间的直播收益总和。

   上述问题可以形式化为：

   给定长度为 `n` 的整数数组 `a`，以及 `q` 次查询。每次查询给出区间 `[l, r]`，请输出该区间元素之和（保证时间复杂度为 **O(1)**，不使用循环语句 ）：
   $$
   a_l + a_{l+1} + \dots + a_r
   $$
   **输入格式**

   第一行：两个整数 `n q`
   第二行：`n` 个整数 `a1 a2 ... an`
   接下来 `q` 行：每行两个整数 `l r`

   **输出格式**

   对每个查询输出一行区间和。

   **输入样例**

   ```
   5 4
   1 2 3 4 5
   1 3
   2 5
   4 4
   1 5
   ```

   **输出样例**

   ```
   6
   14
   4
   15
   ```

   ------

   **提示：**考虑使用数组的前缀和，设：

   - `pre[0] = 0`
   - `pre[i] = a1 + ... + ai`（i 从 1 到 n）

   则查询转化为：

   - `sum(l, r) = pre[r] - pre[l-1]` 

   **思考：**在此基础上，如果每分钟直播的成本为`k`，哪如何选择直播时间段才能使得你最终的总收益最多？

2. 学校里有一台打印机共同学们使用，当任何同学提交一个打印任务时，打印机会先将要打印的内容读入内存，然后进行打印，若将要打印的内容已经在内存中了，打印机就会直接进行打印。但是打印机的内存容量是有限的，因此当打印机内存占满时，打印机会先删除**最早存入内存**的内容，然后再将新内容存入内存并打印。

   现在有N位同学来打印，打印的内容用1-N标号进行表示，相同标号表示内容相同，打印机的内存可以存M个打印内容，你需要计算打印机一共进行了几次存入操作。

   **输入格式**

   共 $2$ 行。每行中两个数之间用一个空格隔开。

   第一行为两个正整数 $M,N$，代表内存容量和打印次数。 

   第二行为 $N$ 个非负整数，按照打印内容的顺序，每个数（大小不超过 $1000$）代表一个打印内容。

   文章中两个打印内容是同一内容，当且仅当它们对应的非负整数相同。

   **输出格式**

   一个整数，为打印机进行存入操作的次数。

   **输入样例**

   ```
   3 7
   1 2 1 5 4 4 1
   ```

   **输出样例**

   ```
   5
   ```

3. 超市里总会有商品录入和数量统计的需求，为了方便管理，商品都会用唯一的商品序列号来标识。你需要维护一个数据结构，用来记录所有商品的数量，并在商品售出与补货时更新他们的库存数量。

   **输入格式**

   第一行，一个正整数 $N$，表示接下来的操作次数

   接下来N行，每行输入一个字符串 x 和一个数字 v，代表一次操作。

   x 代表商品的序列号；当 v > 0 时，代表补货 v 件；v < 0 时代表售出 v件；

   v = 0 时需要输出 x 的库存数量。

   **输出格式**

   每行代表一次输出查询的结果。

   **输入样例**

   ```
   12
   AB12CD34EF56GH78IJ90 50
   AB12CD34EF56GH78IJ90 0
   AB12CD34EF56GH78IJ90 -12
   AB12CD34EF56GH78IJ90 0
   ZX98YU76TS54RQ32PO10 20
   ZX98YU76TS54RQ32PO10 -5
   ZX98YU76TS54RQ32PO10 0
   MN45KL67HJ89GF01DS23 7
   MN45KL67HJ89GF01DS23 -2
   MN45KL67HJ89GF01DS23 0
   ZX98YU76TS54RQ32PO10 0
   QQ11WW22EE33RR44TT55 0
   ```

   **输出样例**

   ```
   50
   38
   15
   5
   15
   0
   ```

4. 一个社交平台支持用户从不同渠道导入“关注列表”（例如：手机通讯录、其他平台同步、历史备份）。由于同一个账号可能在不同渠道中重复出现，系统需要把多份导入结果合并成一份**不重复**的关注名单。每个账号用一个字符串 `id` 表示（可能包含字母、数字、下划线等），当且仅当字符串完全相同，才视为同一个账号。

   你需要按导入顺序处理数据，并输出最终合并后的关注人数。详细描述如下：

   一共有 `K` 份导入列表。每份列表包含若干账号 `id`。合并时需要**去重**，同一个 `id` 只保留一份。

   你只需要输出最终合并后的**不同账号数量**。

   ### 输入格式

   第一行：一个整数 `K`，表示导入列表的份数。
   接下来对于每一份列表：

   - 第一行：一个整数 `m`，表示该列表包含的账号数
   - 第二行：`m` 个字符串，表示账号 `id`

   ### 输出格式

   输出一行，一个整数，表示合并去重后关注账号的总数。

   ### 输入样例

   ```text
   3
   5
   alice bob carl bob dave
   4
   carl eva frank alice
   6
   george eva helen ivan bob bob
   ```

   ### 输出样例

   ```text
   9
   ```
